Appendix C. How to contribute

If you've read this guide this far, you're already part of ooc's history!

If you want to help further, here are some things you can do:

  • Talk about ooc to your friends/colleagues/soulmate/CEO

  • Blog about it / Write an article about it on your website / in your magazine

  • Report bugs/suggestions on launchpad.

  • Send patches to fix bugs/add features (developer access to the git repository will be given out pretty easily)

  • Write documentation: tutorials, etc. And send them for publication to the official website

  • Help testing ooc on various platforms. The main development happens on Linux, but we want to be as portable as possible. Windows/OSX/*BSD/Haiku users, to your test benches!

  • Write a program/game/demo in ooc: show the world that you can use OpenGL/SDL/GTK/etc. libraries with ooc

  • Contribute to rock, the 'ooc compiler written in ooc' effort.

  • Send a kind e-mail to amoswenger@gmail.com to let know you like/use ooc!
